### פענוח "אוטומטי"
- **בידוד / וידאו**: אוטומטי → Replicate אם יש `REPLICATE_API_TOKEN`, אחרת Gemini אם יש `GEMINI_API_KEY`, אחרת Replicate.
- **ראייה / טקסט**: אוטומטי → Gemini אם מוגדר, אחרת Replicate.
- **רקע** (`resolveBackgroundEngine`):
  - קטלוג + לא ביקשו רקע AI → **תמיד פרוצדורלי** (מתעלם מההעדפה).
  - אחרת העדפה מפורשת גוברת; אוטומטי+רקעAI → Gemini אם מוגדר אחרת Replicate.

### מדיניות נפילה (fallback) — `executeWithEngineFallback`
- מדיניות: `"never"` (ברירת מחדל) / `"billing-errors"` / `"always"`.
- נפילה רק כשההעדפה `"auto"`. חריגת מכסה (`QuotaExceededError`) לעולם לא מפעילה נפילה.
- `"billing-errors"` = רק כשהכשל לא-חויב: regex ‏`402|401|insufficient credit|unauthorized|rate limit|billing|payment|out of credits|quota`.
- **בידוד: `"never"`** — מכוון! בידוד Gemini (מסך ירוק) מרסק יהלומים; עדיף כשל ברור.
- **וידאו: `"billing-errors"`** — Kling→Veo רק על כשל לא-מחויב.
- **רקע AI: אין נפילה בכלל** — כשל → שגיאה קשה בעברית (לא רקע פרוצדורלי בשקט).

## 3. סדר פעולות (Pipelines)

### (א) תמונת קטלוג — `pipelineGenerateImage`
1. אימות env + כתובת Cloudinary.
2. **בידוד** — סדר: `cutoutUrl` שסופק → מטמון (L1 זיכרון / L2 DB) → מנוע:
   - Replicate: מקור ב-`w_2048,q_100,f_png` → Bria (`preserve_alpha:true`) → נורמליזציה → העלאה `studio-cutout-bria-*.png`
   - Gemini: data-URI → מסך ירוק → נורמליזציה → `studio-cutout-gemini-*.png`
3. **נורמליזציית בידוד** (`normalizeJewelryCutout`) — שלבים וספים:
   1. הסרת "משבצות שקיפות" (spread≤28, lum≥150, אריחים 8/16/32)
   2. הסרת מסך צבע — flood-fill **מהשוליים בלבד** (דגימת 6% מסגרת, חציון; דורש ≥55% התאמה בשוליים; מרחק צבע <165 או ירוק-דומיננטי; ביטול אם <5% נוקה; despill; ניקוי כתמים בודדים)
   3. אנליזה: opaqueRatio (אלפא>140), bbox, edgeDensity
   4. אם אטימות >0.92 → הסרת רקע בהיר (סובלנות 34, ואז 52) — גם היא flood-fill מהשוליים
   5. **עצירה קשה** אם אטימות >0.85 → שגיאה "הרקע לא הוסר"
   6. ולידציה: אטימות מינימלית 0.008; bbox מינימלי 1%
4. **בידוד פרוצדורלי** (חינם): דורש שוליים אחידים-בהירים (≥92% מהמסגרת: min≥175, spread≤40); שני מעברי strip; נכשל אם אטימות >0.45.
5. **רקע**: פרוצדורלי (Sharp/SVG, קנבס 2048, ספוטלייט 0.42, השחזה 0.2) או AI (בלי נפילה!).
6. **הרכבה** (`compositeProductImage`):
   - cutout בשם `studio-cutout-(bria|gemini)-` → רק הסרת משבצות + ולידציה (בלי נורמליזציה מחדש — היסטורית ריסקה פאסטות!)
   - trim (סף 6) → resize ל-72% מהקנבס (lanczos3; לוידאו ×1.15) → feather אלפא 0.3 (מדולג אם יש אלפא רכה) → הרמוניזציה לפי סגנון → (לוידאו) חידוד עדין sigma 0.45 → fade לשרשרת (אם גובה > רוחב×1.12)
   - מיקום: `PRESET_SURFACE_Y` לכל סגנון (0.56–0.66), היסט אופקי 1.2%
   - צל מגע: רק איפה שנוגע (רצועת 6% תחתונה, אשכולות ≥5% רוחב, סה"כ ≥12%); אטימות 0.28 לסגנונות דרמטיים / 0.18; מוכפל multiply
   - **אין חידוד גלובלי שלישי** (יצר טבעות/הילות)
   - flatten על צבע רקע ממוצע → העלאה `studio-composite-*.png`

### (ב) וידאו שיווקי — `pipelineGenerateVideo`
- **סדר בחירת תמונת בסיס**: תוצאת-תמונה → תצוגה מקדימה → מקור-תמונה → (מקור וידאו:) פריים 0 שטוח (`so_0` + `b_white`).
- פרומפט: LOCK + DEFAULT + רמז-סגנון + הנחיה מתורגמת. ברירת מחדל: 8 שניות (היחיד עם 1080p!), מצב pro.
- **preserve + וידאו מקור**: שיפור Cloudinary חינם (`e_improve,e_denoise:40,e_sharpen:50,vc_h264:high`).
- **preserve + תמונה**: zoompan חינם (`e_zoompan:du_{s};maxzoom_1.02;fps_30`, fl_animated) → הורדה + העלאה מחדש כ-mp4.
- **ai**: Kling (3–15 שניות, מולטי-שוט, אודיו נטיבי) או Veo (4/6/8 שניות; **1080p רק ב-8 שניות ב-pro**, אחרת 720p; polling עד 270 שניות).
- עריכת פוסט (חינם): חיתוך יחס — **`g_center` לוידאו** (לא `g_auto` — גורם 423!); מוזיקה — `l_audio:{id},e_volume,fl_splice,fl_layer_apply` אחרי `ac_none`.

### (ג) וידאו שהועלה
מיטוב חינם / יצירת וידאו AI מפריים ראשון / כלי עריכה חינמיים / שיפור Veo (עם דיאלוג אישור).

### (ד) שיפור מקור — 3 פריסטים Gemini, יחס 1:1, ואז **בידוד חובה מחדש** (הפיקסלים השתנו).

### (ה) מילוי AI לרשימה — ראייה → טקסט; retry אחד על עברית "מתורגמת"; קטגוריה אוטומטית מהראייה.

### (ו) פרסום לקטלוג — ולידציות: כותרת, מחיר ≥0, סוג {natural,lab}, 7 קטגוריות, תמונה מ-Cloudinary בלבד. כל תוצאה נשמרת אוטומטית לספריית התוכן.

## 4. מצבי עבודה ו-State

### StudioV2State (שדות עיקריים)
```
flow: catalog|marketing · source{url,originalUrl,kind,duration} · cutout{url,status,cached}
preview{url,kind,status,presetId} · result{url,kind,status,provider}

aiEngines(auto×5) · productTitle/Price · attempts[≤40] · selectedAttemptId
sourceAdj · resultAspect(original) · videoAdj · usage · error · busyAction · toast
```

### פעולות reducer — מה כל אחת מאפסת
| פעולה | איפוסים |
|---|---|
| SET_FLOW | cutout+preview+result+error |
| SOURCE_UPLOADED | cutout+preview+result (שומר originalUrl ראשון) |
| SET_PRESET | preview בלבד אם הסגנון שונה (result נשמר לפרסום) |
| SET_USE_AI_BACKGROUND | preview+result + סנכרון מנוע רקע |
| SET_ENGINES | סנכרון useAiBackground בקטלוג; אם השתנה → preview+result |
| CUTOUT_DONE | preview+result (בידוד חדש מבטל את מה שנבנה עליו) |
| CONTINUE_FROM_RESULT | תוצאה→מקור חדש, מאפס הכול במורד |
| USE_SOURCE_DIRECTLY | מקור→תוצאה בלי בידוד, provider "original" |

### נעילת busyAction
נעילה גלובלית — פעולה בתשלום אחת בכל רגע; `runPaidAction` מחזיר false אם תפוס; כל הכפתורים disabled.

### גלריית ניסיונות
כל תצוגה/תוצאה/שיפור → attempt (dedupe לפי URL, עד 40). עדיפות תצוגה בקנבס:
ניסיון נבחר → תוצאה → תצוגה מקדימה → בידוד → מקור. בר הפרסום מכבד את הניסיון הנבחר (לא מפרסם תוצאה ישנה).

### דיאלוגי אישור (פעולות יקרות)
וידאו AI (`confirmVideo`) ושיפור Veo (`confirmEnhanceAi`) — עם תג עלות ותצוגה.

### שמירה/שחזור פרויקט
Snapshot v1 כולל `sourceKind` (פרויקטים ישנים → image), auto-save עם debounce ‏1200ms (מדלג בזמן busy). שלבים: 1=אין מקור, 2=יש מקור, 3=בעבודה, 4=יש תוצאה.

## 5. הגנות עלות ותשתית

### Idempotency (`studio_action_locks`)
מפתח לכל כוונה (UUID בצד לקוח); לחיצה כפולה/retry משתמשים באותו מפתח → תוצאה מהמטמון בלי חיוב כפול.
נעילה pending עד 15 דקות → 409; נעילה תקועה → השתלטות. Retry אוטומטי אחד אחרי 1.5 שניות.

### מכסות (`studio_quota_counters`)
UPSERT אטומי `count = count + 1` עם `current_date` של ה-DB (עמיד לעומס אמיתי — נבדק ב-30/5, 100/20).
סקופים: video → `studioDailyVideoLimit`; cutout+background → `studioDailyImageLimit`. release מחזיר את הספירה בכשל.

### מעקב שימוש (`ai_usage_events`)
כל קריאת AI: ספק, יכולת, מודל, מצב, הצלחה, משך, עלות משוערת, cached, projectId. כולל תרגום/טקסט/ראייה (תוקן 9.7).

### מטמון בידוד
L1: Map בזיכרון (TTL שעה). L2: ‏DB דרך `studio_action_locks` (מפתח `cutout:{sha1}`) — שורד deploy.
⚠️ בידוד פגום שנשמר פעם יוגש שוב לנצח עד ניקוי (ר' סעיף 6.3).

### API Routes (`/api/studio/*`) — כולם runtime nodejs + requireAdmin
| Route | maxDuration |
|---|---|
| /generate | 120 |
| /composite | 60 |
| /remove-background | 60 |
| /video | 300 |
| /enhance-source | 120 |
| /enhance-video | 300 |
| /usage (GET) | ברירת מחדל |

שגיאות: idempotency → 409, מכסה → 429, השאר עם דגל `retryable` (לא-retryable: ‏401/402/מכסה/env חסר).

## 6. בעיות ידועות / חוב טכני (לתשומת לב בבנייה החדשה)

1. **חשבון Replicate ריק (402)** — הבידוד נכשל קשה בלי נפילה ל-Gemini (מכוון — הבידוד של Gemini הרסני). עד טעינת קרדיט: בידוד AI חסום; רק פרוצדורלי (רקע אחיד) עובד.
2. **מוזיקת Mixkit חסומה (403)** — 4 סגנונות מנוטרלים (`available:false`). ברירת מחדל בקטלוג-וידאו עדיין `luxury` (חסום) → נופל בשקט להשתקה. פתרון: העלאת קבצים ידנית ל-Cloudinary תחת `yerushalmi-studio/music/*` או יצירה ב-MusicGen (סקריפט מוכן: `scripts/generate-studio-music.mjs`, דורש קרדיט Replicate).
3. **קבצי בידוד ישנים פגומים** — בידודים שנוצרו לפני תיקון ה-flood-fill (9.7) הם קבצים סטטיים הרוסים ב-Cloudinary; פרויקטים שמפנים אליהם צריכים בידוד מחדש. מטמון ה-DB נוקה, אבל ה-URL השמור בפרויקט עצמו לא.
4. **חיווט מנוע-רקע תקין** (בניגוד לחשד) — הספק חי בתוך `aiEngines.background`, אין action נפרד. מוזרות UX: ה-Select לא יכול לבחור "פרוצדורלי" ישירות (הכפתור הראשי עושה זאת).
5. **קוד מת** — `makePreview` (deprecated אך נגיש דרך retry), `MODELS.svd`, `getCachedComposite`, `BASE_JEWELRY_QUALITY`, `DEFAULT_NEGATIVE`, תווית `studio-cutout-local` (אין קוד שמייצר).
6. **שיפור וידאו נרשם לא נכון** — capability ‏"vision" וספירה במכסת **תמונות** במקום וידאו.
7. **מגבלת Veo** — ‏1080p רק ב-8 שניות בדיוק (מצב pro); 5 שניות מעוגל ל-6 (720p). ברירת המחדל שונתה ל-8.
8. **מודל לא מוכר = עלות 0** — שינוי שם מודל ידווח $0 בשקט.
9. **דליפת URL זמני ב-zoompan** — כשל העלאה-מחדש מחזיר URL של טרנספורמציה חיה; עריכות עליו הופכות no-op.

## נספח: לקחים קריטיים לבנייה החדשה (מהבאגים שנפתרו השבוע)

1. **לעולם לא להסיר רקע לפי צבע-פיקסל בודד** — רק flood-fill מהשוליים. פאות יהלום בוהקות = אותו צבע כמו רקע.
2. **לא לבדוק data-URI גדול ברגקס** (`/^data:(.*);base64,(.+)$/` קורס על מגה-בייטים) — רק indexOf/slice.
3. **Cloudinary**: ‏`f_png` בשרשרת טרנספורמציה גובר על סיומת ‎.mp4 (וידאו יוגש כ-PNG); ‏`g_auto` על וידאו → ‏423 אסינכרוני (להשתמש `g_center`).
4. **מכסות**: רק UPSERT אטומי על מונה; לא CTE של insert+count (לא רואים זה את זה); רק זמן DB (`current_date`), לא `new Date()` בצד Node.
5. **כל שינוי מקור מחייב איפוס בידוד — עם הודעה למשתמש** (איפוס שקט נראה כמו באג).
6. **CTA לא נעלם לצמיתות אחרי תוצאה** — חייב לחזור כשמשנים הגדרה רלוונטית.
7. **כל קריאת AI בתשלום** חייבת: reserveStudioQuota + trackAiUsage + idempotency key + תג עלות בממשק.
8. **מנוע "מוגדר" ≠ מנוע "עם קרדיט"** — 402 חייב הודעה ברורה או נפילה בטוחה, לא תוצאה שקטה שבורה.
9. **בידוד cutout שכבר עובד לא מנרמלים שוב** בהרכבה — נורמליזציה כפולה מרסקת פאסטות.
10. **אין fallback הרסני בשקט** — עדיף כשל ברור מתוצאה גרועה שנראית כהצלחה.
